# Email

- [Can I email](https://www.caniemail.com/) (check HTML/CSS support)
- Email over 102KB is clipped and the message `View entire message` is shown at the bottom.

## Template

- `<title>` read by screen readers when email is opened.
- Some email clients strip `<head>`.
- Wrap body contents in `<div>` to provide majority styles.
- Provide inline margin/padding styles for all block-level elements.
- Refer to CAN-SPAM act complaiance for footer disclaimer.
- HTML elements
  - div, span
  - h1 - h6
  - p, strong, em
  - a
    - Do not use relative path.
  - img
    - Do not use relative path.
    - Provide full path with protocol to the hosted web server address. Example `https://kushaj.com/assets/logo.jpg`.
    - Use jpg, png, gif.
    - Provide alternate text only on relevant images.
  - table
- CSS properties
  - Text
    - color
    - font-family
    - font-size
    - font-style
    - font-weight
    - line-height
    - text-align
  - Block level elements
    - margin
    - padding
    - border
    - width
    - max-width
- CSS quirks
  - Provide margin/padding on all block-level elements.
  - Provide font properties on all `<div>` elements.
  - Use `<center>` in addition to `display: block; margin: 0 auto;` trick.
- Buttons
  - Method 1: Bulletproof Email button.
    - [buttons.cm](https://buttons.cm/) to get code. Hard to maintain.
  - Method 2: Nested table.
    - Shown in below example in "Call to action".

## Table Layout

- Add `role="presentation"` to avoid announcing in screen reader.
- Only use `table, tr, td`.
- Override defaults using HTML attributes.
- Place styles on table cells.

```html title="Boilerplate template"
<table border="0" cellpadding="0" cellspacing="0" role="presentation" width="100%">
  <tr>
    <td style=""></td>
  </tr>
</table>
```
