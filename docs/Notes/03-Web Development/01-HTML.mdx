# HTML

## Main root

Tags

- **\<html>**

```html
<!doctype html>
<html lang="en-US" data-theme="dark"></html>
```

## Document metadata

Tags

- **\<head>**
- **\<base>**
- **\<link>**
- **\<meta>**
- **\<style>**
- **\<title>**
  - Search engines show first 55-60 characters.
  - Specify primary purpose of the page, followed by name of the website. Example `Menu - Blue House Chinese Food - FoodYum: Online takeout today!`.
  - During form submission errors, update title to reflect the error. Example `2 errors - Your order - Sea Food Store - Food: Online takeout today!`.
  - Screen readers do not announce title changes, so use _Aria Live Regions_.

Order of elements in _head_ can affect performance. [capo.js](https://rviscomi.github.io/capo.js/) provides weight ordering for _head_ elements.

- [Chrome extension](https://chromewebstore.google.com/detail/capo-get-your-%EF%B9%A4%F0%9D%9A%91%F0%9D%9A%8E%F0%9D%9A%8A%F0%9D%9A%8D%EF%B9%A5/ohabpnaccigjhkkebjofhpmebofgpbeb) - Click extension to run and view results in the console.
- [Online demo](https://rviscomi.github.io/capo.js/user/demo/) - Paste HTML or URL to test.

```html
<!-- Fixes rem not using OS value on mobile. -->
<meta name="text-scale" content="scale" />
<!-- Set color for UI elements like scrollbar.
     Use CSS color-scheme to adjust value on theme change, no JS required. -->

<link rel="manifest" href="manifest.json" />
```

## Content sectioning

### \<address>

- Contact information like name, physical address, URL, email address, phone number, social media handle, geographic coordinates.
- Use to represent contact information of the nearest `<body>` or `<article>` ancestor.
  - Inside `<body>` to provide contact information of the website.
  - Inside `<article>` to provide contact information of the article like author name.

```html
<address>
  You can contact author at
  <a href="http://www.example.com/contact">www.example.com</a>.<br />
  If you see any bugs, please
  <a href="mailto:webmaster@example.com">contact webmaster</a>.<br />
  You may also want to visit us:<br />
  Mountain View, CA 94041<br />
  USA
</address>

<article>
  <footer>
    <address>
      By <a href="">Kushaj</a><br />
      Email: <a href="">alex@semantics-weekly.com</a><br />
      Office: 404 Syntax Lane, Suite 100, Athens, GA
    </address>
  </footer>
</article>
```

### \<article>

- If `<article>` contains `<h1>-<h6>` header tag use `aria-labelledby` to link them.
- Independent item of content, that can be consumed without the context of the website. Like blog post, forum post, news article, product card, user-submitted comment, interactive widget.
- Use `<h1> - <h6>` as child element to help identify the article.
- When nesting `<article>`, the inner article represent content independent of the outer article, like comments on blog post.

```html
<article class="blog-post">
  <h2></h2>
  <section class="comments-section">
    <article class="comment"><h3></h3></article>
    <article class="comment"><h3></h3></article>
  </section>
</article>

<article class="product-card">
  <h3></h3>
  <p></p>
  <button></button>
</article>
```

### \<aside>

- If multiple `<aside>` are present on the page use `aria-label` or `aria-labelledby`.
- Content that is only indirectly related to the main content, and the content can be removed without affecting the webpage.
- Use for sidebar content like trending topics, call-out box, glossary, footnotes, advertising blocks, profile snippet like About the author, related articles, recommended reading.

```html
<article>
  <h1>History of space exploration</h1>
  <aside class="fun-fact-box"></aside>
</article>

<div class="page-layout">
  <main></main>
  <aside class="page-sidebar">
    <h3>Trending topics</h3>

    <h3>Advertisement</h3>
  </aside>
</div>
```

### \<footer>

- Use for the concluding content of the section like
  - Article footer - category tags, author bio, read more links.
  - Component footer - timestamps, reply, flag, report buttons.
  - Aside footer - logal disclaimers for a form, disclosure notes, view all links.
  - Global page footer - copywright notice, links to privacy policy, sitemaps, social icons, contact info.
- Does not have to necessariy appear at the bottom.

### \<header>

- Use under `<body>` to act as banner of the webpage and provide info like logo, name, search feature, global navigation, slogan.
- Use under `<article>, <main>, <section>, <nav>, <aside>` to provide introductory or navigational aids for the surrounding section. Place the `<h1>-<h6>` inside.

```html
<header class="side-header">
  <div class="logo"></div>
  <nav></nav>
</header>

<section class="featured-content">
  <header class="section-header">
    <h2></h2>
    <p>Introductory line</p>
  </header>
</section>

<article class="blog-post">
  <header class="article-header">
    <h1></h1>
    <div class="meta-data"></div>
  </header>
</article>
```

### \<h1> - \<h6>

- Used to construct table of contents information.
  - Users use this to quickly jump from heading to heading in order to determine the contents of the page.
- Do not skip heading levels.
- Every page should have a single `<h1>` to describe content of the page similar to `<title>`.

### \<hgroup>

- To grroup `<h1>-<h6>` with related content.
- Related content can be subheading, alternative title, tagline. Each represented using `<p>` tag.

```html
<hgroup>
  <h1>Title</h1>
  <p>Last updated 12 July 2002</p>
</hgroup>
```

### \<main>

- One per page.
- The content inside `<main>` should be unique to the current document. Content that is repeated like header, footer, sidebar, copywirght info, search forums shouldn't be included.
- Use "skipnav" with main as well.

```html
<body>
  <a href="#main-content">Skip to main content</a>
  <main id="main-content"></main>
</body>
```

### \<nav>

- If multiple `<nav>` are present on the page use `aria-label` or `aria-labelledby`.
- Provide navigation links either within the current document or to other documents, like menus, table of contents, indexes.
- Prose can be used to provide navigation as well.

```html
<header>
  <nav aria-label="Primary navigation">
    <ul>
      <li><a href="">Home</a></li>
    </ul>
  </nav>
</header>

<footer>
  <nav aria-label="Footer navigation"></nav>
</footer>

<nav>
  <p>You are on my home page. To the north lies <a href="">my blog</a>.</p>
  <p>To the south lies a dark and dank <a href="">contacts page</a>.</p>
</nav>
```

### \<section>

- Use `aria-label` or `aria-labelledby` for `<section>`.
- To represent generic standalone section of a document, that does not fit it in other content sectioning tags. Like a list of search results, map display and its controls.
- Most of the times have a heading.
- `<div>` is used as a styling wrapper.

```html
<section aria-labelledby="introduction">
  <h2 id="introduction">Introduction</h2>
  <p></p>
</section>
<section aria-labelledby="criteria">
  <h2 id="criteria">Criteria</h2>
  <p></p>
</section>

<section aria-label="Control Group">
  <button></button>
  <button></button>
</section>
```

### \<search>

- If multiple `<search>` are present on the page use `aria-label` or `aria-labelledby`.
- To represent part of document with a form for search or filtering operation.
  - The search/filtering can be for the entire site or current page.
- Do not use it for presenting search results. They should be part of main contents of the page.
- Suggestions and links part of "Quick search" are parts of the forum and should be present inside `<search>`.

```html
<search>
  <form action="">
    <button>Search</button>
  </form>
</search>

<search>
  <input type="search" />

  <section aria-label="Suggestions and quick links">
    <h3>Results:</h3>
    <ul id="results"></ul>
    <output id="no-results"></output>
  </section>
</search>
```

## Block Text content

### \<blockquote>, \<q>, \<cite>

- Direct quote from an external source.
  - Use `<blockquote>` for block-level element.
  - Use `<q>` for inline quote.
- Source attribution
  - Specify URL to source document using `cite` attribute.
  - Or use `<cite>` to provide text representation of the source. Place it outside `blockquote, q`.
- Use `<cite>` to specify title for creative work like book, paper, song, film, game, website, blog post, comment, social media post, oral statement.
  - This does not include the name of person.

```html
<div>
  <blockquote cite="url">
    <p>Quote</p>
  </blockquote>
  <p>-Author, <cite>Source name</cite></p>
</div>

<p>As <cite></cite> wrote <q cite="url">inline quote.</q></p>
```

### \<dl>, \<dt>, \<dd>

- Use for dispalying key-value pairs information.
  - Glossary.
  - Recipe where ingredient is header and quantity is description.
  - Transcript or conversations, where speaker name is header and spoken words are the description.
- Single term and description (one dt, dd per item).
- Multiple terms, single description (multiple dt, one dd per item).
- Single term, multiple descriptions (one dt, multiple dd per item).
- Each item can be wrapped in `<div>` to style.

```html
<dl>
  <div>
    <dt>Term</dt>
    <dd>Description</dd>
  </div>

  <dt>Term</dt>
  <dt>Term alternative</dt>
  <dd>Description</dd>

  <dt>Term</dt>
  <dd>Description</dd>
  <dd>Description alternative</dd>
</dl>
```

### \<div>, \<span>

- Represent nothing, and is used only to group content and style.

```html
<div>
  <p>Something <span>inline text</span></p>
</div>
```

### \<figure>, \<figcaption>

- To represent self-contained content (with optional caption).
- It can be image, diagram, code snippet, table, poem, quotation that is referenced in the main flow of a docuemnt.
- `<figure>` can be be moved to any part of the document or to appendex without affecting the main flow.

```html
<figure>
  <figcaption></figcaption>
  <img />
</figure>
```

### \<hr>

- Represent a thematic break between paragraph-level elements (change of scene, shift of topic within a section).
- Another way to think about it is as a new section within body of text where it's not necessary to have a heading.

```html
<p></p>

<hr />

<p></p>
```

### \<menu>, \<ol>, \<ul>, \<li>

- `<menu>` - Same as `<ul>`, but the items are interactive.
- `<ol>` - The order matters for the meaning of the content.
  - `reversed` - To reverse the numbering, not the list items.
  - `start="3"`
  - `type="1"` - If you are referencing the items in the HTML document, then use `type` attribute, otherwise use CSS `list-style-type` property.
    - `a` lowercase letter, `A` uppercase letter.
    - `i` lowercase roman, `I` uppercase roman.
    - `1` number.
- `<ul>` - Order of items in the list is meaningless. The items are for display only.

```html
<ol type="i" reversed start="4">
  <li>One</li>
  <!-- To override list numbering -->
  <li value="10">Three</li>
  <li>Four</li>
</ol>

<ul>
  <li>Item</li>
  <li>Item</li>
</ul>

<menu>
  <li><button>Copy</button></li>
  <li><button>Paste</button></li>
</menu>
```

### \<p>

- Structured grouping of related content, such as images, form fields, block of text.
  - Each part of form is considered a paragraph.
- Screen readers provide shortcuts to skip to the next/previous paragraph, letting them skim content.
- Using a block level element inside p tag, will autoclose the p tag.

```html
<form>
  <p>
    <label>Name: <input /></label>
  </p>

  <p>
    <label>Phone: <input type="tel" /></label>
  </p>

  <p>
    <label>Email: <input type="email" /></label>
  </p>

  <fieldset>
    <legend>Size</legend>
    <p>
      <label> <input type="radio" name="size" /> Small </label>
    </p>
    <p>
      <label> <input type="radio" name="size" /> Medium </label>
    </p>
  </fieldset>

  <p><button>Submit</button></p>
</form>

<form>
  <p>Hello. My <label for="firstName">name</label> is <input id="firstName" type="text" />...</p>
</form>
```

### \<pre>

- To show preformatted text. Monospaced font used to display the content.
- The first leading newline character is ignored.
- The content is parsed as HTML.
- If creating illustrations, provide alternate description using `<figure>`, `role`, `aria-label`.
- Escape ambigious characters
  1. Replace `&` with `&amp;`.
  2. Replace `<` with `&lt;`.
- Useful for frontend debugging, instead of relying on `console.log`.

```html
<pre>
  <code>
body {
  color: red;
}
  </code>
</pre>

<figure>
  <pre role="img" aria-label="ASCII COW">
\   ^__^
  \  (oo)\_______
    (__)\       )\/\
        ||----w |
        ||     ||
  </pre>
  <figcaption>The cow is illustrated using preformatted text characters.</figcaption>
</figure>
```

## Inline text content

### \<a>

- Provide URL in `href`, which can include hyperlink, files, email addresses, locations in the same page.
- Content inside should indicate where the link goes.
  - `about our products` more accessible thatn `click here`.
- Assistive software has shortcut to list all links on a page, to quickly scan pages.
- Attributes
  - `downlaod` - The URL is downloaded. `filename` is optional, in which case the browser will suggest a filename/extension.
    - `download="filename.extension"` - Name of downloaded file. `\`, `/` are converted to `_`.
    - Works for same-origin URLs, `blob:`, `data:` schemes.
  - `href`
  - `hreflang`
  - `interestfor` - For Popover API.
  - `ping` - send `POST` request to list of URLs for tracking with the text body `PING`.
